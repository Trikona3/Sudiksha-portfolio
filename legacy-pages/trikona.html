Trikona - Construction Game

Case Study

Project Snapshot

Mission-driven construction bidding and learning simulation

Trikona is a construction game platform with two modes: a bid simulation mode for large real-world-style projects and a learning mode where users practice optimal building decisions.

IND Construction Tech ROLE UX Design · Research · Prototype TOOLS Figma · HTML/CSS · React

Project Description

In bid mode, an admin creates a mission for a large project like a bridge, including location, specs, materials, constraints, and deadline. Bidders then build execution plans and submit budgets based on what they already own vs. what they must source. In learning mode, users practice construction decisions and receive guidance on material choice, structural logic, and build quality.

Design Decisions

Role-based information architecture

Separated admin, bidder, and learner journeys to reduce cognitive load and keep each dashboard action-focused.

Budget realism via resource-aware inputs

Added owned-assets logic (equipment, transport proximity, material inventory) so bids reflect real operational cost.

Mission-first UX for comparability

Structured mission briefs with consistent fields so multiple bidder submissions are easier to compare plan-vs-cost.

Learning feedback built into play loop

Designed instant guidance moments during build tasks, helping users improve decisions while staying in flow.

Outcome

Trikona supports both procurement-style decision making and skills development: admins can evaluate bids on strategy and cost quality, while learners build construction judgment through interactive, high-fidelity scenarios.
